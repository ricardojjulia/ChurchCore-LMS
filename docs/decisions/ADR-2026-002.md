# ADR-2026-002 — Navigation Active State Contract

**Date:** 2026-06-20
**Status:** Accepted
**Deciders:** Council Review 1

## Context

Council Agent 3 found that `aria-current="page"` is missing or inconsistently applied across four navigation components:

| Component | `usePathname()` | `aria-current="page"` | Visual active style |
|---|---|---|---|
| `SidebarClient.tsx` | Yes | **No** | Yes |
| `MobileBottomNav.tsx` | Yes | Yes | Yes |
| `MobileAdminDrawer.tsx` | Yes | **No** | Yes |
| `platform/layout.tsx` NavLink | **No** | **No** | **No** |
| `(reports)/layout.tsx` NavLink | **No** | **No** | **No** |

Screen reader users relying on `aria-current="page"` to understand their location in the platform get inconsistent or absent signals. The mobile bottom nav is the only component that does this correctly.

## Decision

All navigation components in ChurchCore LMS must follow the mobile bottom nav pattern as the canonical implementation:

```tsx
// Required pattern for every nav link
const active = pathname === href || pathname.startsWith(href + '/')

<Link
  href={href}
  aria-current={active ? 'page' : undefined}
  className={cn(baseClass, active && activeClass)}
>
  {children}
</Link>
```

Rules:
1. **`usePathname()`** must be used in every nav component to detect the active route
2. **`aria-current={active ? 'page' : undefined}`** must be set on the active `<Link>` — not `aria-current="true"` or a boolean
3. **Visual active style** must always accompany `aria-current` — they are not optional on each other
4. **`<nav aria-label="...">`** must wrap every navigation region with a descriptive label so screen readers can distinguish sidebar from bottom nav from platform nav

## Files Requiring Update

- `src/components/layout/SidebarClient.tsx` — add `aria-current`
- `src/components/layout/MobileAdminDrawer.tsx` — add `aria-current`
- `src/app/platform/layout.tsx` — add `usePathname()` + `aria-current`
- `src/app/(reports)/layout.tsx` — add `usePathname()` + `aria-current`

## Consequences

- Every new nav component must use this pattern at creation time
- Frontend-builder must include nav active state in the implementation checklist
- Implementation-validator must flag any `<Link>` inside a `<nav>` that lacks `aria-current` handling
