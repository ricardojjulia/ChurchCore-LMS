# COUNCIL-2025-011: Formation Pulse — Weekly AI-Generated Pastoral Nudges

**Status:** RATIFIED  
**Date:** 2026-06-16  
**Council:** Architecture Council, ChurchCore LMS (6/6 unanimous)  
**Tags:** ai, notifications, email, cron, engagement, pastoral, openai

---

## Context

The LMS has full visibility into every student's formation trajectory — XP velocity, progress stalls, cohort standing, days since last engagement, grade trends. This data sits unused between sessions. Every other LMS sends a generic weekly digest. Formation Pulse sends a short, personal, pastor-voiced message generated by the AI from each student's actual data. It is the LMS equivalent of pastoral follow-up at scale.

---

## Decision

Implement Formation Pulse: a weekly cron-triggered system that classifies each active student's trajectory, pre-computes cohort percentile in SQL, builds a minimal non-PII context object, calls OpenAI to generate a 2–3 sentence personal nudge, delivers it via the existing `notifications` table (Realtime) and Resend email, and records the send in a new `formation_nudges` table.

---

## Amendments Adopted

1. **No PII in the OpenAI prompt.** The prompt context must contain only: display name, course titles, XP numbers, level, cohort rank (pre-computed), and days-since-last-access. No email address, auth ID, or uid.
2. **Cohort percentile is pre-computed in Postgres** before the AI call — do not ask the AI to do arithmetic. A SQL query returns `{ rank, total, percentile }` per student per active cohort.
3. **Trajectory classifier is a pure function** (no I/O), separately unit-testable. Input: `{ daysSinceAccess, progressDelta, xpDelta, cohortPercentile }`. Output: `'stalled' | 'active' | 'ahead' | 'completing'`.
4. **`formation_nudges` table rate-gates sends.** If a row exists for a `user_id` with `sent_at` within the last 5 days, skip that student entirely. This prevents double-sends on retry.
5. **Opt-out respects `profiles.email_digest_enabled`.** Students with `email_digest_enabled = false` receive the in-app notification but not the email. The in-app notification cannot be suppressed by this flag (it is not email).
6. **Batch OpenAI calls in groups of 10** using `Promise.allSettled`. A failed generation for one student must never break the batch for others.
7. **`formation_nudges.opened_at`** is nullable; set by a future `/api/nudge/open?id=...` pixel or link tracker. Leave the column in place now for future analytics.

STEP 2 — Trajectory classifier utility
==========================================================================

Create src/lib/formation/trajectoryClassifier.ts:

  export type TrajectoryLabel = 'stalled' | 'active' | 'ahead' | 'completing'

  export interface TrajectoryInput {
    daysSinceAccess:  number   // days since enrollments.last_accessed_at
    progressDelta:    number   // progress_percent increase over last 7 days (0 if unknown)
    xpDelta:          number   // xp_points increase over last 7 days (0 if unknown)
    cohortPercentile: number   // 0–100, pre-computed; 0 if not in any cohort
    progressPercent:  number   // current overall progress (0–100)
  }

  export function classifyTrajectory(input: TrajectoryInput): TrajectoryLabel {
    const { daysSinceAccess, progressPercent, xpDelta, cohortPercentile } = input
    if (progressPercent >= 90) return 'completing'
    if (daysSinceAccess > 10)  return 'stalled'
    if (cohortPercentile >= 75 || xpDelta > 50) return 'ahead'
    return 'active'
  }

No I/O. Pure function. Deterministic. Unit-testable without mocks.
