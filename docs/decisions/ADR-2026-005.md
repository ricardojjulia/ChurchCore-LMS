# ADR-2026-005: @dnd-kit/core for Drag-and-Drop Course Builder

**Date:** 2026-06-21
**Status:** ACCEPTED
**Deciders:** Architecture Council
**Tags:** builder, drag-and-drop, accessibility, dependency

---

## Context

The course block builder (`src/app/courses/[id]/build/`) currently requires keyboard-based reordering
(move up / move down buttons). The Priority 4 feature in the Sprint 2–4 roadmap upgrades this to
mouse/touch drag-and-drop reorder. `course_blocks.sort_order` (FLOAT) already exists and is indexed.

Three library options were evaluated:

**Option A — @dnd-kit/core**: Modular, accessible, no DOM manipulation, works with React state.
Small bundle (~10 KB gzipped core). Maintained. Used widely in production Next.js apps.

**Option B — react-beautiful-dnd**: Older library, archived, no longer maintained. Touch support
requires a separate package. Dropped from consideration.

**Option C — HTML5 Drag and Drop API directly**: Maximum control, zero dependency. Accessibility
requires significant custom work (keyboard equivalents, ARIA live regions). Rejected because the
implementation cost exceeds the benefit given @dnd-kit/core is freely available and accessible by default.

---

## Decision

**@dnd-kit/core** (`@dnd-kit/core` + `@dnd-kit/sortable`) is the drag-and-drop library for the
course block builder. It is added as a production dependency.

The sort order representation remains FLOAT in the database. On drag completion, the frontend
computes new `sort_order` values for the reordered list (midpoint insertion) and calls a server
action `reorderCourseBlocks({ courseId, orderedBlockIds })` which updates all affected rows in a
single transaction.

Keyboard-based move-up / move-down buttons are RETAINED as an accessible fallback. @dnd-kit/core
provides keyboard drag support natively; the explicit buttons remain for users who prefer them.

---

## Consequences

**Enables:**
- Intuitive drag-to-reorder of course blocks in the builder
- Touch device support (teachers building courses on tablets)
- Accessible drag experience via @dnd-kit's built-in keyboard mode

**Constrains:**
- `@dnd-kit/core` and `@dnd-kit/sortable` must be added to `package.json` dependencies
- Bundle size increases by ~15 KB gzipped (acceptable)
- The `reorderCourseBlocks` server action must use the service client (direct UPDATE of sort_order
  bypasses RLS per-row checks — the action verifies org ownership once before updating all rows)
- Drag-and-drop reorder is for course blocks only in this sprint; page/module reorder is out of scope
