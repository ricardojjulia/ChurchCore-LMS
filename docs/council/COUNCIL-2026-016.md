# COUNCIL-2026-016 — Ministry Scenario Support: Enrollment Types, Age Gating & Diploma Tracking

**Date:** 2026-06-24
**Status:** Ratified
**Author:** Platform review (context: demo scenario seed system requiring 5 distinct ministry education models)

---

## Context

The platform needs to seed 6 named demo scenarios (Wed Bible School, Summer Kids School, College Semester, 1-Year Diploma, Ministry Ed Leaders, All Scenarios). Three features are missing that block realistic demos:

1. **Enrollment type** — no way to declare a section as `open` (anyone can self-enroll) vs `cohort_gated` (only cohort members can access). The `course_sections` table has enrollment window dates but no type flag.
2. **Age gating** — no `age_min`/`age_max` on courses. Summer Kids School and similar children's programs cannot declare age restrictions.
3. **Diploma/program completion** — `course_certificates` issues per-course. There is no program-track-level diploma award when a student completes all required courses in a track.

---

## Decisions

**D1:** Add `enrollment_type TEXT NOT NULL DEFAULT 'open' CHECK (enrollment_type IN ('open','cohort_gated','invite_only'))` to `course_sections`. `open` = anyone in the org can self-enroll. `cohort_gated` = only active cohort members can enroll. `invite_only` = explicit enrollment only.

**D2:** Add nullable `age_min SMALLINT` and `age_max SMALLINT` to `courses`. Enforced at application layer (enrollment action checks age against profile DOB when set); displayed in catalog. NOT enforced in RLS (RLS operates on auth, not profile attributes). Add `date_of_birth DATE` to `profiles` if not present.

**D3:** Add `program_diplomas` table — awarded when a student completes all `required` courses in a program track. Trigger fires on `course_certificates` INSERT, checks if the student has now completed all required courses for any track, and inserts into `program_diplomas` if so. A required-course mapping table `program_track_courses` links `program_tracks` to `courses` with a `sequence_order` and `is_required` flag.

### Prompt C — Program Track Course Mapping & Diploma Awards

**Files:**
- `supabase/migrations/20260624130000_program_diplomas.sql` (NEW)
- `src/app/admin/program-tracks/[id]/page.tsx` — add course assignment UI
- `src/app/actions/program-tracks.ts` (NEW) — server actions for managing required courses
- `src/app/api/certificates/[id]/route.ts` — check if a diploma certificate API exists; if so, extend it; otherwise note that diploma display is via the student dashboard
- `src/app/(student)/dashboard/` or `src/app/courses/` — show diploma badge when awarded

**Scope:** Create `program_track_courses` (maps tracks → courses with `sequence_order` and `is_required`) and `program_diplomas` (awarded per student per track). A DB trigger on `course_certificates` INSERT checks if the student has now completed all `is_required` courses for any track they're part of; if so, inserts a `program_diplomas` row (idempotent via UNIQUE constraint). The program track detail page gains a "Required Courses" section where admins can add/remove/reorder courses. The student dashboard shows earned diplomas.

**Work:**
1. Migration:
   ```sql
   CREATE TABLE program_track_courses (
     track_id       UUID NOT NULL REFERENCES program_tracks(id) ON DELETE CASCADE,
     course_id      UUID NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
     sequence_order SMALLINT NOT NULL DEFAULT 0,
     is_required    BOOLEAN NOT NULL DEFAULT TRUE,
     PRIMARY KEY (track_id, course_id)
   );
   ALTER TABLE program_track_courses ENABLE ROW LEVEL SECURITY;
   -- policies: admin/manager all; authenticated read

   CREATE TABLE program_diplomas (
     id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id         UUID NOT NULL REFERENCES profiles(uid) ON DELETE CASCADE,
     track_id        UUID NOT NULL REFERENCES program_tracks(id) ON DELETE CASCADE,
     awarded_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     diploma_no      TEXT UNIQUE NOT NULL DEFAULT ('DIPL-' || UPPER(SUBSTR(gen_random_uuid()::TEXT,1,8))),
     UNIQUE (user_id, track_id)
   );
   ALTER TABLE program_diplomas ENABLE ROW LEVEL SECURITY;
   -- policies: user reads own; staff reads all in org

   -- Trigger: on course_certificates INSERT, check if all required courses in any track are done
   CREATE OR REPLACE FUNCTION public.check_program_completion()
   RETURNS TRIGGER LANGUAGE plpgsql AS $$
   DECLARE
     rec RECORD;
   BEGIN
     FOR rec IN
       SELECT ptc.track_id
       FROM program_track_courses ptc
       WHERE ptc.course_id = NEW.course_id
         AND ptc.is_required = TRUE
     LOOP
       IF NOT EXISTS (
         SELECT 1 FROM program_track_courses ptc2
         WHERE ptc2.track_id = rec.track_id
           AND ptc2.is_required = TRUE
           AND NOT EXISTS (
             SELECT 1 FROM course_certificates cc
             WHERE cc.user_id = NEW.user_id AND cc.course_id = ptc2.course_id
           )
       ) THEN
         INSERT INTO program_diplomas (user_id, track_id)
         VALUES (NEW.user_id, rec.track_id)
         ON CONFLICT (user_id, track_id) DO NOTHING;
       END IF;
     END LOOP;
     RETURN NEW;
   END;
   $$;
   CREATE TRIGGER trg_check_program_completion
     AFTER INSERT ON course_certificates
     FOR EACH ROW EXECUTE FUNCTION public.check_program_completion();
   ```
2. Add "Required Courses" section to program track detail page: list existing required courses with remove button; add-course selector. Server actions: `addCourseToTrack(trackId, courseId, sequenceOrder, isRequired)` and `removeCourseFromTrack(trackId, courseId)`.
3. On student dashboard or profile, show a "Diplomas" section listing `program_diplomas` with diploma number and track name.

**Security:** `program_track_courses` managed by admin/manager only. `program_diplomas` — users read their own; staff reads org-scoped via `current_user_org_id()` join through `program_tracks`. The trigger runs as `SECURITY INVOKER` (default) — it fires under the session that inserted the certificate, which is the student's session via the learning action. Ensure the trigger function only does INSERTs and no RLS-bypassing reads.

**Verification:**
- `npm run typecheck`
- `npm run lint`
- `npm test`
- Manual: create a program track, add 2 required courses, complete both as a student, verify diploma row appears and is shown on dashboard.

### Prompt D — 6-Scenario Demo Seed System

- `src/app/platform/tenants/[id]/ResetActions.tsx` — add scenario picker dropdown
- `src/app/platform/tenants/[id]/DemoLoginButton.tsx` — no changes needed

**Scope:** Replace the single "Reset to Demo" button with a scenario dropdown + button. Six scenarios:
1. **Wednesday Bible School** — `series` term, open enrollment, 2 courses (weekly Bible study format), 1 open cohort, no certificate
2. **Summer School for Kids** — `semester` term (8 weeks), 1 course with `age_min=6, age_max=12`, closed cohort, enrollment window
3. **College Semester** — `semester` term (16 weeks), 3 courses, 1 cohort, prerequisite chain on courses
4. **1-Year Bible Diploma** — `academic_year` term, 4 sequential courses in a program track, diploma tracking, 1 closed cohort
5. **Ministry Ed Leaders** — `self_paced` term (no close date), 3 courses open enrollment, no cohort requirement
6. **All Scenarios** — seeds all 5 above in a single tenant (with prefixed course titles to distinguish)

Each scenario creates: term, courses, cohort (where appropriate), enrollments for demo students, and sets `org.settings.demo.scenario` to the scenario key.

**Work:**
1. Define a `DEMO_SCENARIOS` map keyed by slug, each with a seed function.
2. `resetTenantToDemo(orgId, scenario)` — runs the matching seed; falls back to scenario `'college_semester'` if not provided for backwards compatibility.
3. `ResetActions.tsx` — add a `<select>` for scenario before the confirm input. Selected scenario passed to the server action.
4. Store `scenario` key in `org.settings.demo.scenario` for audit trail.

**Security:** Same as existing `resetTenantToDemo` — platform admin only, service role for all DB writes, `assertPlatformAdmin()` at top of action.
