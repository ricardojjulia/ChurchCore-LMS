## User Story

**As a teacher**, I want to upload a document (PDF, Word, or plain text) or paste raw text,
and have AI generate a structured course outline so that I can bootstrap a new course quickly
without starting from scratch.

**Acceptance Criteria:**
1. The course builder has an "AI Outline" entry point for new courses
2. Teacher can paste text OR upload a file (PDF, .docx, .txt — max 5 MB)
3. The AI returns a structured outline: course title, description, and a list of modules with titles and learning objectives
4. Teacher can accept the outline (creates the course blueprint + blocks) or regenerate
5. Accepted outline populates the course builder with pre-filled blocks (one block per learning objective or module section)
6. No PII or sensitive org data is sent to the AI (only the document content + a system prompt)

**Edge Cases:**
- Uploaded PDF has no extractable text (scanned image) → show error "This PDF appears to be a scanned image. Please use a text-based PDF or paste the content manually."
- Document > 5 MB → rejected before upload ("File too large — maximum 5 MB")
- AI returns malformed JSON → parse error shown; offer retry
- Teacher clicks Accept then navigates away → course blueprint is already created (no orphan prevention in v1)

**Out of Scope:**
- Video transcript processing
- Multi-file upload
- Automatic prerequisite chain generation
- Real-time streaming of outline generation

---

## Gap Being Addressed

Blank-canvas course creation is a significant barrier for non-technical teachers. Most church
teachers have existing curriculum (sermon notes, Bible study guides, books) but lack the technical
skill to structure them into LMS courses. An AI outline generator removes the blank-canvas problem
and dramatically lowers the onboarding time for new content.

## Wildcard Pre-Audit

**PDF text extraction is the hard part.** `@react-pdf/renderer` generates PDFs but does NOT
parse them. PDF text extraction requires a separate library. Options:
1. `pdf-parse` (Node.js, ~30KB) — simple text extraction from text-based PDFs; no scanned image support
2. Send the file to Claude as a base64 attachment (Anthropic's vision API supports PDF documents)

Option 2 (send PDF to Claude directly as a document) is superior: no extraction library needed,
Claude handles the PDF natively, and it works better for mixed content. Use the Anthropic API's
`document` content type with `source.media_type = "application/pdf"` and `source.data = base64_pdf`.

For .docx: the simplest approach is to require teachers to paste the content as text, or to
extract text via a lightweight DOCX parser. For v1, support PDF (via Claude vision) and plain
text (paste). .docx support is a v2 consideration.

**The outline schema matters for downstream population.** The AI must return a JSON structure
that maps directly to `course_blocks`. Design the prompt to elicit:
```json
{
  "course_title": "...",
  "course_description": "...",
  "modules": [
    {
      "title": "Module 1: Introduction",
      "blocks": [
        { "title": "Welcome & Overview", "type": "text", "objective": "..." },
        { "title": "Key Definitions", "type": "text", "objective": "..." }
      ]
    }
  ]
}
```
Each module maps to a section header block; each block maps to a content block. Population
creates the blocks in sequence with `sort_order = 1, 2, 3...`.

**Security: send only document content to Claude.** The prompt must NOT include: org name,
teacher name, user IDs, email addresses. The system prompt is generic ("You are a curriculum
design assistant..."). The user message is just the document content. This is CLAUDE.md compliant
(No PII in AI prompts).

## Positions

### 🏛️ The Architect

New API route: `POST /api/ai/outline-generator`
- Accepts: `{ text?: string, fileBase64?: string, fileType?: string }`
- Validates: max text length 50,000 chars; max base64 size ~7MB (covers 5MB decoded + overhead)
- Calls Anthropic API (Claude) with document content
- Returns: `{ outline: OutlineSchema }`

New page: `src/app/courses/new/outline/page.tsx` (or a modal/drawer in the course builder)
Entry: "Start with AI Outline" button in the new course flow

Population server action: `createCourseFromOutline({ courseId, outline })` — accepts the accepted
outline and creates `course_blocks` rows in sequence.

### 🧭 Product Manager

**Flow:**
1. Teacher on course builder → "✨ Generate outline with AI" button
2. Modal opens: paste text OR upload file
3. "Generate" button → loading spinner (3-8 seconds typical)
4. Outline preview: collapsible module/block list
5. "Accept & Build" button → creates blocks → closes modal → builder shows populated blocks
6. "Regenerate" button → call API again with same content

The Anthropic system prompt:
"You are a curriculum design assistant for religious education. Given document content, extract and structure it into a course outline. Respond with a JSON object matching this schema exactly: { course_title, course_description, modules: [{ title, blocks: [{ title, type, objective }] }] }. module.blocks[].type must be one of: 'text', 'quiz', 'discussion'. ### 🛡️ Security Lead

Critical constraint: **never include org name, teacher name, user IDs, or emails in the AI prompt**. The route handler does NOT fetch any profile or org data for inclusion in the prompt. The system prompt is static. The user message is ONLY the document content.

Rate limiting: apply `ratelimit` (Upstash Redis, from `src/lib/rate-limit.ts`) — 5 requests per teacher per hour. AI inference is expensive; rate limit prevents abuse.

File upload: the base64 file is decoded and validated:
- Check decoded size ≤ 5 MB
- For PDFs: verify magic bytes (`%PDF`) before sending to Claude
- NEVER store the uploaded file; process in memory only

Auth: `getUser()` + role check (teacher/admin/manager only; students cannot call this).

### 🔬 QA Lead

Manual tests:
1. Paste 500 words of Bible study text → outline generated with 3-5 modules → Accept → blocks created in builder
2. Upload 2-page PDF sermon notes → outline generated
3. Upload 6 MB file → rejected before API call
4. AI returns malformed response → error shown, retry available
5. Regenerate → different outline (AI non-deterministic)
6. Student attempts to access endpoint → 403

### 🗄️ Data Engineer

No new tables. Created blocks go into `course_blocks` (existing table).
`createCourseFromOutline` must set:
- `course_id` from the existing course being built
- `block_type_id` = 'text' (or 'section_header' for module titles if that type exists)
- `sort_order` = sequential integers (1, 2, 3...)
- `title` from the outline
- `content` = `{ text: '', objective: block.objective }` — teacher fills in content later

## Council Verdict (6/6 Unanimous)

| Decision | Verdict |
|---|---|
| PDF handling | Send to Claude as document (base64) — no local extraction |
| Text input | Paste text supported; .docx is v2 |
| AI model | claude-sonnet-4-6 (fast, JSON-capable) |
| Rate limiting | 5 requests/hour/teacher via Upstash |
| No PII | Org name, teacher name, user IDs NEVER in prompt |
| Outline population | Server action creates course_blocks from accepted outline |
| File size limit | 5 MB (validated before API call) |
| Storage | No file storage; process in memory |

export async function createCourseFromOutline({
  courseId,
  outline,
}: {
  courseId: string
  outline: OutlineSchema
}): Promise<{ error?: string; blocksCreated?: number }>

Steps:
1. Auth: getUser() + role check (teacher/admin/manager)
2. Verify course.org_id = caller's org_id
3. Check that blockCount = outline.modules.flatMap(m => m.blocks).length ≤ 50 (prevent runaway creation)
4. Build blocks array: flatten modules → blocks with sort_order = index + 1
   - Module title → block with block_type_id = 'text' (or 'section_header' if it exists)
   - Block → block with block_type_id = block.type (check what values are valid)
   - content JSONB: { text: '', objective: block.objective }
5. Service client: INSERT INTO course_blocks (course_id, org_id, block_type_id, title, content, sort_order)
6. revalidatePath(`/courses/${courseId}/build`)
7. Return { blocksCreated: blocks.length }
