## User Story

**As a learner**, I want to enter Focus Mode while studying so that distracting navigation,
sidebars, and headers are hidden and I can concentrate on the course content.

**Acceptance Criteria:**
1. A "Focus Mode" toggle button is visible in the learn/course page
2. When activated: main nav, sidebar, and course header are hidden; only the content area and a minimal top bar with the toggle remain
3. Focus mode preference is persisted in localStorage (survives page refresh)
4. Exiting focus mode restores the full layout
5. Keyboard shortcut to toggle: `F` key (not in input fields)

**Edge Cases:**
- Mobile: focus mode already has limited chrome — on screens < 768px, focus mode only hides the nav (sidebar is already collapsed)
- Screen reader: the hidden nav elements use `aria-hidden="true"` (not `display: none`) so they're still accessible if needed, OR use actual CSS class toggle with `hidden` — either is acceptable; prefer the approach used elsewhere in the codebase
- User with cookies/localStorage disabled → focus mode defaults to off; no error

**Out of Scope:**
- Server-side persistence of focus mode preference
- Per-course focus mode settings
- Pomodoro timer integration

---

## Gap Being Addressed

The learn page (`src/app/courses/[id]/`) renders a full navigation shell with sidebar, header,
and course navigation. For learners who want immersive study sessions, there is no way to minimize
the chrome. This is a low-effort, high-value UX feature — the only state needed is a single boolean
in localStorage.

---

## Wildcard Pre-Audit

**Pure CSS + React context is sufficient.** No Supabase, no server action, no migration.
Add a `useFocusMode()` hook that reads/writes localStorage and provides a `[isFocusMode, toggle]`
tuple. Wrap the learn page layout with a context provider. In the layout component, conditionally
add a CSS class (e.g., `data-focus-mode="true"` on `<html>` or `<body>`) that hides nav elements
via CSS selectors (e.g., `[data-focus-mode="true"] .site-nav { display: none }`).

**The `F` keyboard shortcut must not fire in inputs.** Check `event.target` before toggling:
if `event.target instanceof HTMLInputElement || event.target instanceof HTMLTextAreaElement`,
ignore the keypress.

**Identify the exact elements to hide.** Read the learn page layout before writing code.
The layout may be: a shell layout at `src/app/(auth)/layout.tsx` or similar, or the course
page itself. Find the nav and sidebar elements and note their identifiers (CSS class, id,
or structural position in the component tree).

### 🧭 Product Manager

**Toggle button**: floating button in the bottom-right of the learn page. Icon: 🔲 (expand) in normal mode, ✕ or 🔳 (compress) in focus mode. Tooltip: "Enter Focus Mode" / "Exit Focus Mode".

**Focus mode visual**: everything except the content card and the toggle button fades out or disappears. Simple CSS transition (opacity: 0 or display: none on nav/sidebar) is sufficient.

### ⚙️ The Engineer

New hook: `src/hooks/useFocusMode.ts`
```typescript
'use client'
import { useState, useEffect } from 'react'
const KEY = 'churchcore_focus_mode'
export function useFocusMode() {
  const [isFocusMode, setIsFocusMode] = useState(false)
  useEffect(() => {
    setIsFocusMode(localStorage.getItem(KEY) === 'true')
  }, [])
  const toggle = () => setIsFocusMode(prev => {
    const next = !prev
    localStorage.setItem(KEY, String(next))
    return next
  })
  return [isFocusMode, toggle] as const
}
```

New component: `src/components/ui/FocusModeToggle.tsx`

- Returns the toggle button

In the learn page: add `FocusModeToggle` to the page. Apply conditional CSS classes to nav/sidebar based on `isFocusMode`. Read the learn page layout first to understand what needs to be hidden and how to pass the state down.

'use client'
import { useState, useEffect, useCallback } from 'react'
const STORAGE_KEY = 'churchcore_focus_mode'

export function useFocusMode(): [boolean, () => void] {
  const [isFocusMode, setIsFocusMode] = useState(false)

  useEffect(() => {
    try {
      setIsFocusMode(localStorage.getItem(STORAGE_KEY) === 'true')
    } catch {
      // localStorage unavailable — default to false
    }
  }, [])

  const toggle = useCallback(() => {
    setIsFocusMode(prev => {
      const next = !prev
      try { localStorage.setItem(STORAGE_KEY, String(next)) } catch {}
      return next
    })
  }, [])

  return [isFocusMode, toggle]
}

=== PHASE C — FocusModeToggle Component ===

File: src/components/ui/FocusModeToggle.tsx

'use client'
import { useEffect } from 'react'
import { useFocusMode } from '@/hooks/useFocusMode'

export function FocusModeToggle() {
  const [isFocusMode, toggle] = useFocusMode()

  useEffect(() => {
    function onKeyDown(e: KeyboardEvent) {
      if (e.key !== 'f' && e.key !== 'F') return
      if (e.target instanceof HTMLInputElement) return
      if (e.target instanceof HTMLTextAreaElement) return
      if ((e.target as HTMLElement).isContentEditable) return
      e.preventDefault()
      toggle()
    }
    window.addEventListener('keydown', onKeyDown)
    return () => window.removeEventListener('keydown', onKeyDown)
  }, [toggle])

  return (
    <button
      onClick={toggle}
      className="fixed bottom-6 right-6 z-50 p-3 rounded-full bg-white border border-border shadow-lg text-sm font-semibold text-muted-foreground hover:text-foreground transition-all"
      aria-label={isFocusMode ? 'Exit Focus Mode' : 'Enter Focus Mode'}
      title={`${isFocusMode ? 'Exit' : 'Enter'} Focus Mode (F)`}
    >
      {isFocusMode ? '⊠' : '⊡'}
    </button>
  )
}
