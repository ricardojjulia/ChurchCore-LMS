# COUNCIL-2026-008: PDF Certificate Download

**Tags:** certificates, pdf, react-pdf, download

## User Story

**As a learner**, I want to download a PDF certificate of completion for any course I've finished
so that I can share proof of my learning with my church, ministry, or employer.

**As an admin**, I want certificates to display the church/org name and the learner's full name
so that the certificate is credible and branded to our ministry.

**Acceptance Criteria:**
1. A "Download Certificate" button appears on completed courses (where `course_certificates` row exists)
2. Clicking the button generates and downloads a PDF with: learner name, course title, org name, issued date, certificate number, final grade (if available)
3. The PDF is generated server-side to prevent tampering
4. The certificate number matches `course_certificates.certificate_no` (CERT-XXXXXXXX format)
5. PDF renders cleanly on A4 and US Letter paper sizes

**Edge Cases:**
- No certificate yet (course not completed) → button is hidden; "Complete the course to earn your certificate" message
- `@react-pdf/renderer` is already installed — no new dependency
- Grade is null → omit grade line from PDF; don't show "null%"

**Out of Scope:**
- Custom branding (logo, colors) per org — use default ChurchCore branding in Sprint 2
- Email delivery of certificate
- Verifiable certificate URL for third parties

---

## Gap Being Addressed

`course_certificates` table exists and `issue_certificate()` function works. But there is no
PDF generation, no download button, and no certificate template. The data is there; the UI and
PDF renderer are missing.

---

## Wildcard Pre-Audit

**Generate server-side, return binary.** @react-pdf/renderer has two modes: browser-side
`<PDFViewer>` (renders in an iframe) and server-side `renderToBuffer()` (returns `Buffer`).
For a download endpoint, `renderToBuffer()` is correct — generate in a Route Handler, return
with `Content-Disposition: attachment; filename="certificate.pdf"`.

**Do NOT stream the PDF from a Supabase Storage bucket.** That would require storing generated
PDFs, managing storage lifecycle, and dealing with signed URL expiry. Instead generate on-demand.
The data is a few rows; generation is fast (< 500ms).

**The certificate number is the canonical identifier.** Display `CERT-XXXXXXXX` prominently.
This is what a verifier would use if we ever build a verification page.

**Typography matters.** Church certificates need to feel formal. Use the built-in @react-pdf/renderer
fonts or embed a single Google Font subset. A clean serif + sans-serif pairing is appropriate.
Use `Font.register()` for any non-system font.

### ⚙️ The Engineer

`src/app/api/certificates/[id]/pdf/route.ts`:
```typescript
export const runtime = 'nodejs' // renderToBuffer requires Node.js
export async function GET(req: NextRequest, { params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return Response.json({ error: 'Unauthorized' }, { status: 401 })

  // Fetch certificate — RLS ensures user owns it
  const { data: cert } = await supabase
    .from('course_certificates')
    .select('*, courses(title, org_id), organizations(name)')
    .eq('id', id)
    .single()

  if (!cert) return Response.json({ error: 'Not found' }, { status: 404 })

  const pdfBuffer = await renderToBuffer(<CertificateDocument ... />)
  return new Response(pdfBuffer, {
    headers: {
      'Content-Type': 'application/pdf',
      'Content-Disposition': `attachment; filename="certificate-${cert.certificate_no}.pdf"`,
    },
  })
}
```

CertificateDocument uses `@react-pdf/renderer` primitives: `Document`, `Page`, `Text`, `View`, `StyleSheet`.

Download button: a simple `<a href={/api/certificates/${id}/pdf} download>` link styled as a button.
No JS needed — native browser download.

### 🛡️ Security Lead

The Route Handler MUST authenticate via `createClient().auth.getUser()`.
The certificate query relies on RLS: `course_certificates` SELECT policy allows `user_id = current_user_uid()`.
This means a student can only download their own certificates — RLS enforces this; no extra check needed.

`runtime = 'nodejs'` is mandatory — `renderToBuffer` is not Edge-compatible.

Do not log the certificate ID or learner name to stdout. Log only errors.

### 🔬 QA Lead

Manual tests:
1. Completed learner clicks download → PDF downloads with correct name, course, cert number
2. Incomplete learner (no certificate row) → button not shown
3. Another user's certificate ID → 404 (RLS blocks the fetch)
4. Course with no grade → PDF renders without grade line

### 🗄️ Data Engineer

No migration needed. `course_certificates` already has all required fields:
`user_id, course_id, issued_at, final_grade, letter_grade, certificate_no`
(from migration 20240601000028_xp_and_certificates.sql)

`course_certificates` → `courses` → `organizations` chain is joinable in Supabase select syntax.
`courses.org_id` → `organizations.id`.

=== PHASE B — Route Handler ===

File: src/app/api/certificates/[id]/pdf/route.ts

export const runtime = 'nodejs'

export async function GET(
  req: NextRequest,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return Response.json({ error: 'Unauthorized' }, { status: 401 })

  // RLS on course_certificates ensures this user owns the cert
  const { data: cert, error } = await supabase
    .from('course_certificates')
    .select(`
      id, certificate_no, issued_at, final_grade, letter_grade,
      courses ( title, org_id ),
      profiles!course_certificates_user_id_fkey ( display_name )
    `)
    .eq('id', id)
    .single()

  if (error || !cert) return Response.json({ error: 'Certificate not found' }, { status: 404 })

  // Type-cast join results (Supabase inference limitation)
  const course = cert.courses as { title: string; org_id: string } | null
  const profile = cert.profiles as { display_name: string } | null

  // Fetch org name separately
  const { data: org } = await supabase
    .from('organizations')
    .select('name')
    .eq('id', course?.org_id ?? '')
    .single()

  const { renderToBuffer } = await import('@react-pdf/renderer')
  const { CertificateDocument } = await import('@/components/pdf/CertificateDocument')
  const React = await import('react')

  const buffer = await renderToBuffer(
    React.createElement(CertificateDocument, {
      learnerName: profile?.display_name ?? 'Learner',
      courseTitle: course?.title ?? 'Course',
      orgName: org?.name ?? 'ChurchCore',
      issuedAt: cert.issued_at,
      certificateNo: cert.certificate_no,
      finalGrade: cert.final_grade,
      letterGrade: cert.letter_grade,
    })
  )

  return new Response(buffer, {
    headers: {
      'Content-Type': 'application/pdf',
      'Content-Disposition': `attachment; filename="certificate-${cert.certificate_no}.pdf"`,
      'Cache-Control': 'private, no-cache',
    },
  })
}
